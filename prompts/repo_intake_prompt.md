# Repo Intake Prompt

Lies das Zielrepo zuerst als System, nicht als lose Dateimenge.

Wichtig:
- neue Funde zuerst als Intake behandeln, nicht direkt als kuratierte Wahrheit
- Intake-Dossier und Projektbindung lesen, bevor die Landkarte aktualisiert wird
- GitHub-Metadaten, README und Aktivitaetssignale vor dem Review mitlesen

Arbeite in dieser Reihenfolge:

1. Zweck des Repos verstehen
2. Hauptartefakte und Hauptfluss erkennen
3. aktive Kernlogik von Nebenlogik trennen
4. offensichtliche Schichten benennen
5. erste potenzielle externe Vergleichsmuster notieren

Wichtig:
- kein blinder Lösungsvorschlag
- keine vorschnellen Architektururteile
- zuerst Repo lesen, dann abstrahieren

Antwortformat:
- Repo-Zweck
- Kernfluss
- relevante Schichten
- erste externe Analogien
- offene Fragen
